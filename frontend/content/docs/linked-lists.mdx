# Linked Lists

A linked list is a fundamental linear data structure consisting of a sequence of nodes. Unlike arrays, linked lists do not store elements in contiguous memory locations. Instead, each element (node) contains two parts:
1. The **data** itself.
2. A **pointer** (or reference) to the next node in the sequence.

This dynamic structure allows for efficient insertions and deletions at any point in the list, but it sacrifices the ability to access elements directly by index.

## Real-world Analogy

Think of a linked list like a **scavenger hunt**. 
You are given the first clue (the head node). When you solve it, it doesn't give you all the clues at once; it just tells you where to find the *next* clue (the pointer). You must follow each clue one by one in order to reach the end of the hunt (the null pointer).

<br />

## Why use Linked Lists?

When you already have arrays, why do you need linked lists?

**Advantages over Arrays:**
- **Dynamic Size:** Linked lists can grow and shrink dynamically during runtime. You don't need to define an initial size.
- **Fast Insertions/Deletions:** Inserting or deleting a node, especially at the beginning of the list, is an $O(1)$ operation (if you already have a pointer to the location).
- **No Memory Waste:** Memory is allocated only when required, so there's no unused capacity.

**Disadvantages:**
- **No Random Access:** You cannot access an element directly (e.g., `list[3]`). You must traverse from the head node, making access $O(N)$.
- **Extra Memory:** Every node requires extra memory to store the pointer(s).
- **Cache Unfriendly:** Since nodes are scattered in memory, linked lists don't benefit from CPU cache locality like contiguous arrays do.

<br />

## Types of Linked Lists

### 1. Singly Linked List
The most basic form. Each node points only to the next node. The last node points to `null`.
```text
[Data|Next] -> [Data|Next] -> [Data|Next] -> null
```

### 2. Doubly Linked List
Each node has two pointers: one to the next node and one to the previous node. This allows traversal in both directions.
```text
null <- [Prev|Data|Next] <-> [Prev|Data|Next] -> null
```

### 3. Circular Linked List
The last node points back to the first node instead of `null`, forming a circle.
```text
┌─────────────────────────────────┐
↓                                 │
[Data|Next] -> [Data|Next] -> [Data|Next]
```

<br />

## Time & Space Complexity

### Time Complexity

| Operation | Best Case | Average Case | Worst Case | Notes |
| :--- | :--- | :--- | :--- | :--- |
| **Access** | $O(1)$ | $O(N)$ | $O(N)$ | Must traverse from the head. |
| **Search** | $O(1)$ | $O(N)$ | $O(N)$ | Linear search is required. |
| **Insert (head)** | $O(1)$ | $O(1)$ | $O(1)$ | Simply update the new node's next pointer and head. |
| **Insert (tail)** | $O(1)$ | $O(N)$ | $O(N)$ | $O(1)$ if you maintain a tail pointer, otherwise $O(N)$ to find the end. |
| **Delete (head)** | $O(1)$ | $O(1)$ | $O(1)$ | Update head to `head.next`. |
| **Delete (tail)** | $O(1)$ | $O(N)$ | $O(N)$ | Need to find the second-to-last node to update its pointer. |

### Space Complexity
- **Overall:** $O(N)$ to store $N$ elements.
- **Per Node overhead:** $O(1)$ extra space per node for the pointer (or two pointers in a doubly linked list).

Here is how you implement a basic Singly Linked List from scratch.

<CodeTab language="C++" title="LinkedList.cpp">
```cpp
#include <iostream>

// Define the Node structure
struct Node {
    int data;
    Node* next;
    
    Node(int val) : data(val), next(nullptr) {}
};

class LinkedList {
private:
    Node* head;

public:
    LinkedList() : head(nullptr) {}

    // Insert at the beginning: O(1)
    void insertFront(int val) {
        Node* newNode = new Node(val);
        newNode->next = head;
        head = newNode;
    }

    // Insert at the end: O(N)
    void insertEnd(int val) {
        Node* newNode = new Node(val);
        
        if (head == nullptr) {
            head = newNode;
            return;
        }
        
        Node* current = head;
        while (current->next != nullptr) {
            current = current->next;
        }
        current->next = newNode;
    }

    // Delete a node by value: O(N)
    void deleteValue(int val) {
        if (head == nullptr) return;
        
        // If head needs to be deleted
        if (head->data == val) {
            Node* temp = head;
            head = head->next;
            delete temp;
            return;
        }
        
        Node* current = head;
        while (current->next != nullptr && current->next->data != val) {
            current = current->next;
        }
        
        // If value was found
        if (current->next != nullptr) {
            Node* temp = current->next;
            current->next = current->next->next;
            delete temp;
        }
    }

    // Print the list: O(N)
    void print() {
        Node* current = head;
        while (current != nullptr) {
            std::cout << current->data << " -> ";
            current = current->next;
        }
        std::cout << "null\n";
    }
};
```

## Common Operations: Reversing a Linked List

Reversing a linked list is one of the most common interview questions. The algorithm works by iterating through the list and changing the `next` pointer of each node to point to the `previous` node.

### Step-by-Step Dry Run

Suppose we have: `1 -> 2 -> 3 -> null`

1. **Initialization:**
   - `prev = null`
   - `current = head` (which is node 1)
   - `next = null`

2. **Iteration 1 (current = 1):**
   - Save next: `next = current.next` (node 2)
   - Reverse pointer: `current.next = prev` (node 1 now points to null)
   - Advance prev: `prev = current` (prev is now node 1)
   - Advance current: `current = next` (current is now node 2)
   - *State:* `1 -> null`, `2 -> 3 -> null`

3. **Iteration 2 (current = 2):**
   - Save next: `next = current.next` (node 3)
   - Reverse pointer: `current.next = prev` (node 2 now points to node 1)
   - Advance prev: `prev = current` (prev is now node 2)
   - Advance current: `current = next` (current is now node 3)
   - *State:* `2 -> 1 -> null`, `3 -> null`

4. **Iteration 3 (current = 3):**
   - Save next: `next = current.next` (null)
   - Reverse pointer: `current.next = prev` (node 3 now points to node 2)
   - Advance prev: `prev = current` (prev is now node 3)
   - Advance current: `current = next` (current is now null)
   - *State:* `3 -> 2 -> 1 -> null`

5. **End of Loop:** `current` is `null`. We return `prev` (node 3) as the new head.

```python
def reverse_list(head):
    prev = None
    current = head
    
    while current is not None:
        next_node = current.next  # 1. Save the next node
        current.next = prev       # 2. Reverse the pointer
        prev = current            # 3. Move prev forward
        current = next_node       # 4. Move current forward
        
    return prev  # New head
```

## Two-Pointer Technique (Fast and Slow Pointers)

Another fundamental pattern for linked lists is the **Tortoise and Hare** algorithm (Fast and Slow pointers). 

By moving one pointer at 1x speed (`slow`) and another at 2x speed (`fast`), you can solve several tricky problems:
1. **Finding the middle node:** When `fast` reaches the end, `slow` is exactly at the middle.
2. **Detecting cycles:** If there's a loop in the list, `fast` will eventually lap `slow` and they will point to the same node.

```python
def has_cycle(head):
    slow = head
    fast = head
    
    while fast and fast.next:
        slow = slow.next          # Move 1 step
        fast = fast.next.next     # Move 2 steps
        
        if slow == fast:
            return True           # Cycle detected
            
    return False                  # No cycle
```

## Common Pitfalls & Mistakes

> [!WARNING]
> **Losing the Head**
> 
> Never use your `head` pointer to traverse the list! Always create a temporary `current` pointer (e.g. `curr = head`). If you reassign `head`, you lose a reference to the beginning of your list, effectively deleting it.

> [!CAUTION]
> **Null Pointer Exceptions**
> 
> Always check if `current` AND `current.next` are null before accessing their properties. Trying to access `current.next.next` when `current.next` is null will crash your program.
