## Definition

A trie, or prefix tree, stores one character per edge. Nodes represent prefixes and terminal markers represent complete words.

## Intuition

Words with common beginnings share nodes, so prefix queries follow one path instead of comparing against every stored word.

## Real-world analogy

Autocomplete narrows suggestions after every keystroke: typing ca reaches a shared prefix before branching to cat, car, and can.

## Visual explanation

~~~text
root
 └─ c ─ a ─┬─ t*
           ├─ r*
           └─ n*
* marks a complete word
~~~

## Time complexity

| Operation | Time |
| --- | --- |
| Insert word of length L | O(L) |
| Search word | O(L) |
| Starts-with prefix | O(L) |
| Enumerate matches | O(L + output) |

## Space complexity

| Structure | Space |
| --- | --- |
| Trie | O(total characters stored) |
| Dense alphabet child arrays | O(nodes × alphabet size) |

## Implementations

### C++

~~~cpp
#include <unordered_map>
using namespace std;

struct Trie {
  bool word = false;
  unordered_map<char, Trie*> next;
  void insert(const string& text) {
    Trie* node = this;
    for (char ch : text) {
      if (!node->next[ch]) node->next[ch] = new Trie();
      node = node->next[ch];
    }
    node->word = true;
  }
};
~~~

### Java

~~~java
import java.util.*;

class Trie {
  boolean word;
  Map<Character, Trie> next = new HashMap<>();
  void insert(String text) {
    Trie node = this;
    for (char ch : text.toCharArray())
      node = node.next.computeIfAbsent(ch, ignored -> new Trie());
    node.word = true;
  }
}
~~~

### Python

~~~python
class Trie:
    def __init__(self) -> None:
        self.next: dict[str, Trie] = {}
        self.word = False

    def insert(self, text: str) -> None:
        node = self
        for char in text:
            node = node.next.setdefault(char, Trie())
        node.word = True
~~~

## Dry run

Insert cat, creating c -> a -> t*. Insert car; c and a already exist, so only branch r* is added. Searching ca is a prefix match but not a word match.

## Common mistakes

- Confusing a prefix node with a completed word.
- Allocating a full alphabet array when sparse maps are more suitable.
- Omitting deletion cleanup and leaving unreachable branches.

## Advanced notes

Compressed radix trees merge one-child paths to save memory. A ternary search trie can trade speed for lower per-node overhead, while Aho-Corasick adds failure links for multi-pattern matching.
