# Graphs

A **Graph** is a non-linear data structure consisting of vertices (often called nodes) and edges (lines connecting those nodes). While a tree represents hierarchical relationships, a graph represents a network of relationships where any node can connect to any other node.

In fact, a tree is just a special type of graph: a connected, acyclic, undirected graph.

## Real-world Analogy

Think of a **Social Network** or a **City Map**.
- On Facebook, you (a vertex) are friends with someone else (an edge). If you are friends with them, they are friends with you (undirected).
- On Twitter/X, you (a vertex) follow a celebrity (an edge). They don't necessarily follow you back (directed).
- On Google Maps, cities are vertices, and the highways between them are edges. The length of the highway is the "weight" of the edge.

<br />

## Graph Terminology

- **Vertex (V):** A node in the graph.
- **Edge (E):** A connection between two vertices.
- **Directed Graph (Digraph):** Edges have a direction (A -> B).
- **Undirected Graph:** Edges are bidirectional (A &lt;-> B).
- **Weighted Graph:** Edges have a value or cost associated with them.
- **Connected Graph:** There is a path between every pair of vertices.
- **Cycle:** A path that starts and ends at the same vertex without traversing any edge twice.
- **Degree:** The number of edges connected to a vertex (In-degree and Out-degree for directed graphs).

## Representing a Graph

There are two primary ways to represent a graph in code. Suppose we have a graph with 3 nodes: 0, 1, and 2. Node 0 connects to 1 and 2. Node 1 connects to 2.

### 1. Adjacency Matrix
A 2D array of size $V \times V$ where $matrix[i][j]$ is `1` if there is an edge from vertex $i$ to vertex $j$, and `0` otherwise.
- **Pros:** Checking if an edge exists is $O(1)$.
- **Cons:** Uses $O(V^2)$ space, which is terrible for sparse graphs (graphs with few edges).

```text
  0 1 2
0 0 1 1
1 0 0 1
2 0 0 0
```

### 2. Adjacency List (Most Common)
An array of lists (or a hash map). The index represents the vertex, and the list contains all vertices it connects to.
- **Pros:** Space efficient $O(V + E)$. Fast to iterate over all neighbors of a vertex.
- **Cons:** Checking if a specific edge exists takes $O(D)$ where $D$ is the degree of V.

```text
0 -> [1, 2]
1 -> [2]
2 -> []
```

## Graph Traversal

Just like trees, we can traverse graphs using Depth-First Search (DFS) or Breadth-First Search (BFS). 
**Crucial Difference:** Because graphs can have cycles, we MUST keep track of which nodes we have already visited using a `visited` set or array to prevent infinite loops.

### Depth-First Search (DFS)
Explores as far as possible along each branch before backtracking. Great for pathfinding, topological sorting, and cycle detection.

<CodeTabs>
<CodeTab language="C++" title="GraphDFS.cpp">
```cpp
#include <iostream>
#include <vector>

void dfsHelper(int node, const std::vector<std::vector<int>>& adj, std::vector<bool>& visited) {
    visited[node] = true;
    std::cout << node << " ";
    
    // Visit all neighbors
    for (int neighbor : adj[node]) {
        if (!visited[neighbor]) {
            dfsHelper(neighbor, adj, visited);
        }
    }
}

void dfs(int startNode, const std::vector<std::vector<int>>& adj) {
    std::vector<bool> visited(adj.size(), false);
    dfsHelper(startNode, adj, visited);
}
```
</CodeTab>
<CodeTab language="Python" title="graph_dfs.py">
```python
def dfs(start_node, adj_list):
    visited = set()
    
    def dfs_helper(node):
        visited.add(node)
        print(node, end=" ")
        
        for neighbor in adj_list[node]:
            if neighbor not in visited:
                dfs_helper(neighbor)
                
    dfs_helper(start_node)
```
</CodeTab>
</CodeTabs>

### Breadth-First Search (BFS)
Explores the neighbor nodes first, before moving to the next level neighbors. Ideal for finding the **shortest path** in an unweighted graph.

<CodeTabs>
<CodeTab language="C++" title="GraphBFS.cpp">
```cpp
#include <iostream>
#include <vector>
#include <queue>

void bfs(int startNode, const std::vector<std::vector<int>>& adj) {
    std::vector<bool> visited(adj.size(), false);
    std::queue<int> q;
    
    visited[startNode] = true;
    q.push(startNode);
    
    while (!q.empty()) {
        int node = q.front();
        q.pop();
        std::cout << node << " ";
        
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) {
                visited[neighbor] = true;
                q.push(neighbor);
            }
        }
    }
}
```
</CodeTab>
<CodeTab language="Python" title="graph_bfs.py">
```python
from collections import deque

def bfs(start_node, adj_list):
    visited = set([start_node])
    queue = deque([start_node])
    
    while queue:
        node = queue.popleft()
        print(node, end=" ")
        
        for neighbor in adj_list[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```
</CodeTab>
</CodeTabs>

## Implicit Graphs

In many interview questions, you won't be explicitly given a graph with an adjacency list. Instead, you'll be given a **2D Matrix (Grid)**. 
In a grid, each cell is a vertex, and its edges connect to its up, down, left, and right neighbors.

**Example: Number of Islands**
Given an $m \times n$ 2D binary grid which represents a map of '1's (land) and '0's (water), return the number of islands.

```python
def numIslands(grid):
    if not grid: return 0
    
    rows, cols = len(grid), len(grid[0])
    islands = 0
    
    def dfs(r, c):
        # Base case: out of bounds or water
        if r < 0 or c < 0 or r >= rows or c >= cols or grid[r][c] == "0":
            return
            
        # Mark as visited by sinking the island
        grid[r][c] = "0"
        
        # Visit all 4 neighbors
        dfs(r+1, c)
        dfs(r-1, c)
        dfs(r, c+1)
        dfs(r, c-1)
        
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1":
                islands += 1
                dfs(r, c) # Sink the entire island
                
    return islands
```

## Common Interview Questions

1. **Number of Islands:** The ultimate introduction to implicit grid graphs and DFS.
2. **Clone Graph:** A deep copy problem that tests your understanding of object references and cycle prevention.
3. **Course Schedule:** Determines if you can finish all courses given prerequisites. (Topological Sort / Cycle Detection).
4. **Rotting Oranges:** A classic multi-source BFS problem to find the minimum time until all oranges rot.
5. **Word Ladder:** Find the shortest transformation sequence from a start word to an end word (BFS).
