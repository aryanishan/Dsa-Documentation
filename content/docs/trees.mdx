# Trees

Unlike arrays, linked lists, stacks, and queues, which are linear data structures, a **tree** is a hierarchical data structure. 
It consists of nodes connected by edges. A tree has one root node, and every other node is connected by exactly one incoming edge from another node (its parent).

## Real-world Analogy

Think of a **Family Tree** or a **Corporate Organization Chart**. 
The CEO is at the top (the root). The CEO manages several VPs. Each VP manages Directors, who manage Managers, and so on. No one has two direct bosses, and there are no circular reporting lines.

<br />

## Tree Terminology

- **Node:** An entity that contains a value and pointers to child nodes.
- **Root:** The topmost node in the tree.
- **Parent:** A node that has branches extending to other nodes.
- **Child:** A node connected underneath a parent.
- **Leaf:** A node with no children (at the bottom of the tree).
- **Edge:** The connection between two nodes.
- **Depth:** The number of edges from the root to a specific node.
- **Height:** The number of edges on the longest path from a specific node to a leaf.

<br />

## Binary Trees

The most common type of tree in computer science is the **Binary Tree**. In a binary tree, every node has at most two children, commonly referred to as the `left` child and `right` child.

```text
       A         <-- Root
     /   \
    B     C      <-- Internal Nodes
   / \     \
  D   E     F    <-- Leaf Nodes
```

<br />

## Tree Traversal

Because trees are non-linear, there are multiple ways to visit all the nodes. The three standard **Depth-First Search (DFS)** traversals are:

1. **Pre-order (Root, Left, Right):** `A -> B -> D -> E -> C -> F`
   *(Useful for cloning the tree or printing structure.)*
   
2. **In-order (Left, Root, Right):** `D -> B -> E -> A -> C -> F`
   *(Crucial for Binary Search Trees, as it visits nodes in sorted order.)*
   
3. **Post-order (Left, Right, Root):** `D -> E -> B -> F -> C -> A`
   *(Useful for deleting the tree, as you delete children before parents.)*

Additionally, there is **Breadth-First Search (BFS)** or **Level-order traversal**:
- **Level-order (Top to bottom, left to right):** `A -> B -> C -> D -> E -> F`

class BinaryTree {

    // In-order traversal: Left -> Root -> Right
    void inorder(TreeNode* root) {
        if (root == nullptr) return;
        
        inorder(root->left);
        std::cout << root->val << " ";
        inorder(root->right);
    }
    
    // Calculate the maximum depth (height) of the tree
    int maxDepth(TreeNode* root) {
        if (root == nullptr) return 0;
        
        int leftDepth = maxDepth(root->left);
        int rightDepth = maxDepth(root->right);
        
        return std::max(leftDepth, rightDepth) + 1;
    }
};

## Problem Solving Pattern: Depth-First Search (DFS)

Recursion is your best friend when dealing with trees. Most tree problems can be solved by thinking about the problem from the perspective of a single node.

**The Magic Question:** *"If I know the answer for my left subtree, and I know the answer for my right subtree, how do I calculate the answer for my current node?"*

**Example: Invert a Binary Tree**
If you have a function that successfully inverts the left subtree, and successfully inverts the right subtree, what must the current node do? It just needs to swap its `left` and `right` pointers!

```python
def invertTree(root):
    if not root:
        return None
        
    # 1. Ask the recursion fairy to invert the children
    left_inverted = invertTree(root.left)
    right_inverted = invertTree(root.right)
    
    # 2. Swap them at the current node
    root.left = right_inverted
    root.right = left_inverted
    
    return root
```

## Problem Solving Pattern: Breadth-First Search (BFS)

When a problem asks for things "level by level" or "shortest path", use a Queue to perform a Breadth-First Search.

```python
from collections import deque

def levelOrder(root):
    if not root: return []
    
    res = []
    queue = deque([root])
    
    while queue:
        level_size = len(queue)
        current_level = []
        
        # Process all nodes on the current level
        for _ in range(level_size):
            node = queue.popleft()
            current_level.append(node.val)
            
            if node.left: queue.append(node.left)
            if node.right: queue.append(node.right)
            
        res.append(current_level)
        
    return res
```

## Common Interview Questions

1. **Maximum Depth of Binary Tree:** The "Hello World" of tree problems.
2. **Invert Binary Tree:** A classic DFS swapping problem.
3. **Same Tree:** Traverse two trees simultaneously and compare values.
4. **Binary Tree Level Order Traversal:** The ultimate BFS template problem.
5. **Lowest Common Ancestor:** Find the lowest node that has both nodes as descendants.
